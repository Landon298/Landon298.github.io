前言

今天来讲一些关于web方面的题目，需要用到的工具有burpsuite,AntSword等，干货满满，如果有不懂的地方欢迎在评论区提问。

1.[极客大挑战 2019]Http

题目信息

  • 类别： Web
  • 题目描述： 无

解题思路、步骤：

首先启动靶机，打开后发现页面是一个安全技术小组的介绍

然后直接ctrl+u打开源代码，仔细观察后有一个.php文件，

直接打开，发现页面展示如下：

它的意思是提示你不是来自https://Sycsecret.buuoj.cn，然后想到题目Http，提示我们可以通过抓包来修改http报文header,于是直接抓包（我用的是burpsuit内置的浏览器），并且在header中添加**Referer:https://Sycsecret.buuoj.cn**，用来伪造访问来源

放包后在响应包的ptrtty（高亮）栏目发现了提示

具体意思是让我们使用Syclover 浏览器，但是我们可以通过修改User-Agent协议来伪造访问工具为 Syclover 浏览器，于是直接对其进行修改（重新抓包）

再次放包后发现页面再次出现提示

具体意思是这只能在本地阅读，于是在header添加X-Forwarded-For:127.0.0.1，用来伪造本地再次访问，这时得到flag

附文

HTTP 头部字段：
Server:服务器指纹
Set-Cookie:向浏览器端设置Cookie
Last-Modified:服务器通过这个消息告诉浏览器，资源的最后修改时间
Content-Length:正文的长度
Refresh:服务器通过这个Refresh头告诉浏览器定时刷新浏览器
Location:服务器返回给客户端，用于重定向到新的位置
Vary:标识不可缓存的请求头列
Connection：链接方式
Host:请求的web服务器域名地址
User-Agent：HTTP客户端运行的浏览器类型的详细信息
Accept:指定客户端能够接收的内容类型。
Accept-Language：指客户端浏览器用来展示返回信息所选择的语言
cookie: HTTP请求发送时，会把保存在该请求域名下的所有cookie值一起发送给web服务器。
Referer:包含一个URL，用户从该URL代表的页面出发访问当前请求的页面
X-Forwarded-For:是一个HTTP扩展头部，用来表示HTTP请求端真实IP。
Content-Type:用于指定MIME类型，告诉浏览器如何解析响应体。

2.[极客大挑战 2019]Upload

题目信息

  • 类别： Web
  • 题目描述： 无

解题思路、步骤：


首先启动靶机，页面提示我们上传头像，根据这种题的常见思路：先上传一句话木马，然后再进行抓包，修改header中的content-type，最后直接用蚁剑连接，就可以看到flag了。那么步骤如下：
1.上传一句话木马，<? php $eval($_POST['cmd']); ?>，但是上传后发现这种不行，然后换成phtml格式再继续尝试（phtml：是一个嵌入了php脚本的html页面），phtml格式的一句话木马为：


2.上传后页面发生改变

然后进行抓包，由于我们上传的是文件而不是图片，所以我们需要更改content-type为image/jpeg，从而绕过。参考其他师傅的wp说这里的php格式不行，需要一直试到phtml才可以（绕过后缀的文件格式有php,php3,php4,php5,phtml.pht）。

最后放包，得到新的页面

这说明我们已经上传成功了。最后得到flag文件有两种方法；

在网页中得到flag（推荐）

1.在靶机地址那个页面尝试上传一张图片，发现该页面是在
upload文件夹中，于是直接访问upload文件夹，即靶机地址/upload,然后我们就进入到了该文件夹，这时后只需要打开我们上传的那个文件就会得到flag了。

使用蚁剑连接得到flag（主要是复习一下使用方法）

打开蚁剑，右键添加数据，在url地址一栏填入靶机地址/upload/filename.phtml,（filename意思是你上传的.phtml文件的名字），然后点击测试连接，如果连接成功，就点击连接，就可以渗透进去了，进去后直接打开终端，输入cat /flag，就可以得到flag了。

注：

一、使用Burp Suite / Proxy 拦截修改从而伪造content-type绕过检测：
1.正常上传 .phtml 文件，浏览器默认发送：
2.用 Burp Suite 拦截请求，修改 Content-Type 为 image/jpeg：
3.如果服务器仅检查 Content-Type，可能绕过检测。

二、.phtml文件解释

@eval($_POST[shell]);
1.eval() 是 PHP 的危险函数，可以执行任意代码。
2.$_POST[shell] 表示攻击者可以通过 POST 请求传递 shell 参数并执行任意命令（如 shell=system(‘whoami’)）。
3.@ 用于抑制错误信息（避免被发现）。
4.system('cat /flag');
system() 直接执行系统命令，这里尝试读取 /flag 文件（常见于 CTF 题目）。
